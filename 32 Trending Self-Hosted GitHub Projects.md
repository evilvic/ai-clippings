Today, I’ve got 32 trending self-hosted projects from GitHub.

![](https://www.youtube.com/watch?v=_j_GisoWXVk)

No. 1 - Keeper.sh

![](https://githubawesome.com/content/images/2026/03/image-374.png)

Managing three separate calendars across Google, Outlook, and iCloud means double-bookings are basically inevitable. Keeper is a self-hosted sync tool that bridges all three, pulling events together so your availability is accurate everywhere. It also ships as an MCP server, so your local AI agents can read and manage your schedule directly.

[https://github.com/ridafkih/keeper.sh](https://github.com/ridafkih/keeper.sh?ref=githubawesome.com)

No. 2 - Grimmory

![](https://githubawesome.com/content/images/2026/03/image-375.png)

Grimmory is a self-hosted library manager for EPUBs, PDFs, and comics including CBZ and CBR formats. Built-in browser reader, automatic metadata fetching, and reading progress sync directly to Kobo e-readers and OPDS-compatible apps. Your entire digital library in one place, with no Amazon account required to access it. For anyone who's been meaning to get their books off Kindle and onto their own hardware, this is the clean solution.

[https://github.com/grimmory-tools/grimmory](https://github.com/grimmory-tools/grimmory?ref=githubawesome.com)

No. 3 - ministack

![](https://githubawesome.com/content/images/2026/03/image-376.png)

LocalStack pulling core services behind a paywall hit a lot of local development setups hard. MiniStack is the free replacement. It emulates 20 AWS services in a single Docker container, but instead of mocking API responses it spins up real Postgres databases, Redis instances, and runs SQL through DuckDB. 30 megabytes of RAM, boots in under two seconds. Point your existing code at port 4566 and nothing else changes.

[https://github.com/Nahuel990/ministack](https://github.com/Nahuel990/ministack?ref=githubawesome.com)

No. 4 - Sync-in Server

![](https://githubawesome.com/content/images/2026/03/image-377.png)

Sync-in Server is a self-hosted alternative to Google Drive and Dropbox that you deploy on your own infrastructure. File storage, syncing, team collaboration, fine-grained access controls, multi-user support. Modern interface, full ownership of your data. For anyone who's been meaning to get their documents off someone else's servers, this is a complete solution rather than a partial one.

[https://github.com/Sync-in/server](https://github.com/Sync-in/server?ref=githubawesome.com)

No. 5 - Ichinichi

![](https://githubawesome.com/content/images/2026/03/image-378.png)

Most note-taking apps give you infinite flexibility and somehow make it harder to actually write anything. Ichinichi gives you one note per day. Once the day ends, that note is locked permanently. No going back to clean it up, reframe it, or second-guess it. Everything stores locally with AES-GCM encryption. No folders, no tags, no organizational system to procrastinate inside.

[https://github.com/katspaugh/ichinichi](https://github.com/katspaugh/ichinichi?ref=githubawesome.com)

No. 6 - RetroAssembly

![](https://githubawesome.com/content/images/2026/03/image-379.png)

RetroAssembly is a self-hosted retro game cabinet that runs in the browser. Spin it up with Docker, point it at your ROMs, and it handles the rest. Emulators stream directly to desktop or phone, game art pulls automatically, and save states sync across devices. Your entire retro library, accessible from anywhere, on hardware you actually own.

[https://github.com/arianrhodsandlot/retroassembly](https://github.com/arianrhodsandlot/retroassembly?ref=githubawesome.com)

No. 7 - Apprise

![](https://githubawesome.com/content/images/2026/03/image-380.png)

Running multiple self-hosted services means juggling multiple notification APIs, each with their own setup and authentication. Apprise replaces all of that with a single library. One line of code or a simple webhook sends alerts to Discord, Slack, Telegram, SMS, and 80+ other platforms simultaneously. For any serious home lab setup, this is the notification layer that should already be in the stack.

[https://github.com/caronc/apprise](https://github.com/caronc/apprise?ref=githubawesome.com)

No. 8 - Kula

![](https://githubawesome.com/content/images/2026/03/image-381.png)

Prometheus and Netdata are powerful tools that carry a lot of weight for monitoring a single VPS. Kula is a single compiled binary with no external dependencies and no database. It reads directly from the kernel's /proc directory and serves a real-time metrics dashboard in the browser. Drop it on a server, run it, done. Monitoring that matches the scale of the problem.

[https://github.com/c0m4r/kula](https://github.com/c0m4r/kula?ref=githubawesome.com)

No. 9 - ghgrab

![](https://githubawesome.com/content/images/2026/03/image-382.png)

Cloning a 5GB repository to get two files out of it is one of those small inefficiencies that adds up to real wasted time. Ghgrab opens an interactive terminal UI from a GitHub link, lets you browse the repo, and downloads only the files you actually need. One command, pick what you want, skip the rest.

[https://github.com/abhixdd/ghgrab](https://github.com/abhixdd/ghgrab?ref=githubawesome.com)

No. 10 - Ziit

![](https://githubawesome.com/content/images/2026/03/image-383.png)

WakaTime is useful but your coding habits are surprisingly personal data to be handing to a third party. Ziit is a self-hosted alternative. Install the VS Code or JetBrains extension, and it tracks languages, editors, branches, and coding hours silently in the background. The analytics dashboard runs from your own server. Same metrics, none of the data leaving your machine.

[https://github.com/0PandaDEV/Ziit](https://github.com/0PandaDEV/Ziit?ref=githubawesome.com)

No. 11 - Homescreen-Hero

![](https://githubawesome.com/content/images/2026/03/image-384.png)

Plex homescreens go stale fast when the same movies sit in the same collections for weeks. Homescreen-Hero rotates and curates your Plex collections on a schedule, pulling custom lists from Trakt and Letterboxd automatically. There's also an admin dashboard for streaming analytics and content management. For anyone running Plex for a household or friend group, this is the curation layer the app never shipped with.

[https://github.com/trentferguson/homescreen-hero](https://github.com/trentferguson/homescreen-hero?ref=githubawesome.com)

No. 12 - TRIP

![](https://githubawesome.com/content/images/2026/03/image-385.png)

Planning a trip across multiple Google Docs and spreadsheets is a mess that somehow became the standard approach. TRIP is a self-hosted web app that lets you drop points of interest onto an interactive map and build day-by-day itineraries in one place. Runs in a Docker container, no telemetry, no ads, location data stays on your own hardware.

[https://github.com/itskovacs/trip](https://github.com/itskovacs/trip?ref=githubawesome.com)

No. 13 - SkillHub

![](https://githubawesome.com/content/images/2026/03/image-386.png)

Managing custom skills and tools across multiple AI agents with no central registry means the same capabilities get rebuilt or drift out of sync across different projects. SkillHub from iFLYTEK is a self-hosted registry for agent skills. Publish, version, and share reusable AI tools across your organization the same way you'd manage packages or container images. Role-based access control built in, deployable on-premise with Docker.

[https://github.com/iflytek/skillhub](https://github.com/iflytek/skillhub?ref=githubawesome.com)

No. 14 - NOMAD

![](https://githubawesome.com/content/images/2026/03/image-387.png)

TRIP handles the minimalist case well. NOMAD is for when you're planning with a group and need the full stack. Real-time WebSocket collaboration so multiple people can edit the itinerary simultaneously, multi-currency budget splitting, interactive maps, automated packing lists, and a PTO tracker built in. The whole trip planning workflow, self-hosted on your own server.

[https://github.com/mauriceboe/NOMAD](https://github.com/mauriceboe/NOMAD?ref=githubawesome.com)

No. 15 - OpenReview

![](https://githubawesome.com/content/images/2026/03/image-388.png)

Most AI code review tools require sending your source code to someone else's servers. For proprietary codebases, that's a hard no for a lot of teams. OpenReview from Vercel Labs is self-hosted. Deploy it on your own infrastructure, connect it as a private GitHub App, and it reviews pull requests automatically, catches bugs, and commits formatting fixes directly to the branch.

[https://github.com/vercel-labs/openreview](https://github.com/vercel-labs/openreview?ref=githubawesome.com)

No. 16 - Rudel

![](https://githubawesome.com/content/images/2026/03/image-389.png)

Most people running Claude Code have no real visibility into what their sessions are actually costing until the bill arrives. Rudel hooks into your terminal, uploads transcripts to a local ClickHouse database, and generates dashboards tracking token usage, session durations, and exact model costs. Self-hosted, local, and finally answers the question of where all those tokens are actually going.

[https://github.com/obsessiondb/rudel](https://github.com/obsessiondb/rudel?ref=githubawesome.com)

No. 17 - Walrus

![](https://githubawesome.com/content/images/2026/03/image-390.png)

Most autonomous agent setups come with a Python environment, a list of dependencies, and an afternoon of configuration. Walrus is a single compiled binary. Built-in local LLM inference, persistent SQLite memory, and multi-channel support for Terminal, Discord, and Telegram out of the box. No cloud dependencies, no complex setup. The leanest possible path to a self-hosted AI worker that actually runs.

[https://github.com/openwalrus/walrus](https://github.com/openwalrus/walrus?ref=githubawesome.com)

No. 18 - Freeciv Longturn Server

![](https://githubawesome.com/content/images/2026/03/image-391.png)

Longturn is a multiplayer Civilization-style game where each turn takes 23 real hours to complete. The server setup is straightforward enough, but the detail worth paying attention to is the automated cron job that runs an LLM after each turn to generate an in-universe newspaper. Wars, alliances, and diplomatic events from the previous turn, summarized as period-appropriate news coverage, ready when players log in for the next one.

[https://github.com/ndroo/freeciv.andrewmcgrath.info](https://github.com/ndroo/freeciv.andrewmcgrath.info?ref=githubawesome.com)

No. 19 - MikroDash

![](https://githubawesome.com/content/images/2026/03/image-392.png)

The MikroTik Winbox interface has looked the same since most current developers were in middle school. MikroDash is a self-hosted web dashboard that replaces it with real-time graphs for bandwidth, CPU load, and active client connections. Same router, considerably better visibility into what it's actually doing. Network monitoring that doesn't require squinting at a UI from 2003.

[https://github.com/SecOps-7/MikroDash](https://github.com/SecOps-7/MikroDash?ref=githubawesome.com)

No. 20 - TorchCode

![](https://githubawesome.com/content/images/2026/03/image-393.png)

Calling ML APIs teaches you to use ML. TorchCode teaches you how it actually works. It's a self-hosted LeetCode-style practice environment for PyTorch, deployed via Docker. Implement core neural network operators from scratch, attention mechanisms, softmax, and more, with instant auto-grading and feedback on every attempt. For anyone serious about going deeper than the API surface, this is the practice environment worth grinding through.

[https://github.com/duoan/TorchCode](https://github.com/duoan/TorchCode?ref=githubawesome.com)

No. 21 - StayView

![](https://githubawesome.com/content/images/2026/03/image-394.png)

Property management software subscriptions are expensive for what most small hosts actually need. StayView pulls reservations directly from Booking.com and other APIs into a single self-hosted dashboard. Real guest names, booking details, and calendar availability in one clean interface, running on your own hardware. For bed and breakfast owners and small-scale hosts, this is the management layer without the monthly fee attached to it.

[https://github.com/louisnelza/stayview](https://github.com/louisnelza/stayview?ref=githubawesome.com)

No. 22 - Workout Tracker

![](https://githubawesome.com/content/images/2026/03/image-395.png)

Most fitness apps want your health data as much as they want your subscription fee. Workout Tracker is a self-hosted Flask and SQLite app that logs exercises, tracks reps, and monitors personal records from a clean mobile-friendly interface. Deploy it with Docker, and your training data stays on your own hardware. Your gains, nobody else's business.

[https://github.com/enrique-paulino/workout-tracker](https://github.com/enrique-paulino/workout-tracker?ref=githubawesome.com)

No. 23 - Malhaus

![](https://githubawesome.com/content/images/2026/03/image-396.png)

MyAnimeList going down or breaking their API has derailed enough watch lists to make self-hosting worth considering. Malhaus is a self-hosted anime and manga tracker with a modern web UI that matches the polish of the cloud alternatives. Your watch history, ratings, and read progress on your own hardware, with no dependency on a third-party service staying online.

[https://github.com/toorandom/malhaus](https://github.com/toorandom/malhaus?ref=githubawesome.com)

No. 24 - OpenPhotos

![](https://githubawesome.com/content/images/2026/03/image-397.png)

Google Photos changing their storage tiers was the nudge a lot of people needed to finally find an alternative. OpenPhotos is a self-hosted photo library designed for scale, running on your own NAS. Fast thumbnail generation, date and album organization, and no corporate server scanning your family photos for ad targeting.

[https://github.com/openphotos-ca/openphotos](https://github.com/openphotos-ca/openphotos?ref=githubawesome.com)

No. 25 - Atomic

![](https://githubawesome.com/content/images/2026/03/image-398.png)

Most note-taking apps store your thoughts. Atomic connects them. It takes your Markdown notes, automatically chunks and embeds them using semantic similarity, and surfaces connections you didn't explicitly create. Powered by sqlite-vec for fast vector search, with a force-directed canvas that lets you see how your ideas actually relate to each other. Your notes as a living knowledge graph rather than a folder of files you search through manually.

[https://github.com/kenforthewin/atomic](https://github.com/kenforthewin/atomic?ref=githubawesome.com)

No. 26 - Foldergram

![](https://githubawesome.com/content/images/2026/03/image-399.png)

Browsing local photo backups through a file explorer is a genuinely bad experience for something that should feel good to revisit. Foldergram indexes your folders and serves them as an Instagram-style scrolling feed, locally. Fast previews and thumbnails generated via SQLite and FFmpeg, no cloud sync, no external APIs.

[https://github.com/foldergram/foldergram](https://github.com/foldergram/foldergram?ref=githubawesome.com)

No. 27 - TuneLog

![](https://githubawesome.com/content/images/2026/03/image-400.png)

Spotify's recommendation engine is genuinely good, but it requires feeding your listening habits to Spotify to get there. TuneLog brings that same idea to your self-hosted music library. It monitors what you replay and what you skip, builds a taste profile over time, and generates recommendations from your own collection. No manual ratings, no data leaving your machine.

[https://github.com/adiiverma40/tunelog](https://github.com/adiiverma40/tunelog?ref=githubawesome.com)

No. 28 - Obsidian Headless Sync Docker

![](https://githubawesome.com/content/images/2026/03/image-401.png)

Obsidian Sync works well but the monthly fee adds up for what is essentially file synchronization. This Docker image lets you self-host the Obsidian LiveSync backend on your own server with a single Docker Compose command. Real-time, end-to-end encrypted sync across phone, laptop, and tablet, running on your own infrastructure for free. Same sync, zero subscription.

[https://github.com/Belphemur/obsidian-headless-sync-docker](https://github.com/Belphemur/obsidian-headless-sync-docker?ref=githubawesome.com)

No. 29 - Kumiho

![](https://githubawesome.com/content/images/2026/03/image-402.png)

Kumiho is a self-hosted reader for comics, manga, webtoons, and e-books supporting CBZ, EPUB, and PDF. It reads directly from your existing folder structure without database scans, which makes setup nearly instant. The standout feature is the audio support: drop an MP3 into a comic's folder and Kumiho plays it as background music while you read that chapter.

[https://github.com/aha-hyeong/kumiho](https://github.com/aha-hyeong/kumiho?ref=githubawesome.com)

No. 30- X(PFeRD

![](https://githubawesome.com/content/images/2026/03/image-403.png)

German e-invoicing compliance is genuinely painful to get right, and most accounting software charges a premium for handling it. X(PFeRD is a self-hosted web app with a WYSIWYG PDF editor that generates invoices meeting XRechnung and ZUGFeRD standards, including the legally required embedded XML files. Custom font support, Docker deployment, and no monthly subscription.

[https://github.com/tiehfood/xpferd](https://github.com/tiehfood/xpferd?ref=githubawesome.com)

No. 31 - CollabMD

![](https://githubawesome.com/content/images/2026/03/image-404.png)

Collaborating on Markdown files through Git means merge conflicts the moment two people touch the same document. CollabMD turns your local Markdown folders and Obsidian vaults into a real-time collaborative web app. Host it locally, invite your team, and everyone edits the same file simultaneously with live cursors, exactly like Google Docs, with all data staying on your own server.

[https://github.com/andes90/collabmd](https://github.com/andes90/collabmd?ref=githubawesome.com)

No. 32 - Heorot

![](https://githubawesome.com/content/images/2026/03/image-405.png)

Heorot is a Discord-style Matrix client with drop-in voice channels powered by LiveKit. Same experience as jumping into a Discord voice channel, running entirely on your own Matrix homeserver. For anyone moving a community off Discord and onto infrastructure they actually control, this closes the UX gap that usually makes the migration painful.

[https://github.com/Vitosicz/Heorot](https://github.com/Vitosicz/Heorot?ref=githubawesome.com)
