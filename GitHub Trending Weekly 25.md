**No. 1 — droidclaw** [*https://github.com/unitedbyai/droidclaw*](https://github.com/unitedbyai/droidclaw?ref=githubawesome.com)

![](https://githubawesome.com/content/images/2026/02/image-253.png)

You know those old Android phones sitting in a drawer? droidclaw turns them into AI agents. Give it a goal in plain English — it reads the screen, reasons about what to do, taps, types, and swipes via ADB until the job is done. No APIs needed — just install your favorite apps on the device and let the agent work through them like a human would. It comes with 35 ready-to-use workflows for messaging, and lifestyle tasks.

**No. 2 — micasa** [*https://github.com/cpcloud/micasa*](https://github.com/cpcloud/micasa?ref=githubawesome.com)

![](https://githubawesome.com/content/images/2026/02/image-254.png)

micasa is a TUI for home ownership. Tracks maintenance schedules with auto-computed due dates, projects from planning to completion, vendor quote comparisons, appliance warranty tracking, incident logging, file attachments, and spending analytics.Single SQLite file on your machine—no cloud, no account, no subscription. It writes SQL, runs the query, and summarizes results. Vim-style modal keys, fuzzy search, column sorting.

**No. 3 — Surge** [*https://github.com/surge-downloader/surge*](https://github.com/surge-downloader/surge?ref=githubawesome.com)

![](https://githubawesome.com/content/images/2026/02/image-255.png)

Surge is a download manager built in Go for the terminal. Opens multiple connections simultaneously with three optimizations: splits the largest chunk when a worker goes idle, uses work-stealing so fast connections help slow ones, and restarts any worker below 30% of mean speed for a better connection path. Three modes: interactive TUI dashboard, headless background server, and CLI client for scripting and batch downloads.

**No. 4 — BarraCUDA** [*https://github.com/Zaneham/BarraCUDA*](https://github.com/Zaneham/BarraCUDA?ref=githubawesome.com)

![](https://githubawesome.com/content/images/2026/02/image-256.png)

Someone looked at NVIDIA's walled garden and thought, "how hard can it be?" BarraCUDA is an open-source CUDA compiler targeting AMD GPUs—written in fifteen thousand lines of C99, with zero LLVM dependency. You feed it the same.cu files you'd give nvcc, and it compiles directly to AMD RDNA 3 GFX11 machine code, spitting out ELF.hsaco binaries that AMD GPUs can actually run.

**No. 5 — NullClaw** [*https://github.com/nullclaw/nullclaw*](https://github.com/nullclaw/nullclaw?ref=githubawesome.com)

![](https://githubawesome.com/content/images/2026/02/image-257.png)

The arms race for the leanest AI agent infrastructure just got a new contender, and this one is written in Zig. NullClaw is a 678 kilobyte static binary that boots in under 2 milliseconds on Apple Silicon, under 8 milliseconds on a 0.8 GHz edge core, and uses about 1 megabyte of RAM at peak. Everything is a vtable interface — swap providers, channels, tools, memory backends, sandbox and tunnel providers with a config change.

**No. 6 — AssppWeb** [*https://github.com/Lakr233/AssppWeb*](https://github.com/Lakr233/AssppWeb?ref=githubawesome.com)

![](https://githubawesome.com/content/images/2026/02/image-258.png)

AssppWeb is a self-hosted tool for installing iOS apps outside the App Store. The server doesn't receive Apple credentials—all Apple API communication happens in the browser via WebAssembly using libcurl with Mbed TLS.The server acts as a TCP relay using the Wisp protocol and handles IPA compilation from CDN downloads. The developers warn against using public instances, since a malicious host could modify the frontend to capture credentials.

**No. 12 — x86CSS** [*https://github.com/rebane2001/x86CSS*](https://github.com/rebane2001/x86CSS?ref=githubawesome.com)

![](https://githubawesome.com/content/images/2026/02/image-264.png)

x86CSS is an x86 CPU emulator written entirely in CSS. It runs 8086 machine code compiled from C using GCC.The implementation uses CSS animations and container queries to clock the CPU without JavaScript. It includes a C preprocessor, 8086 instruction set, memory model, and I/O system, all in CSS. A Python script generates repetitive CSS sections.The demo runs a C program that executes computation within CSS.

**No. 16 — enveil** [*https://github.com/GreatScott/enveil*](https://github.com/GreatScott/enveil?ref=githubawesome.com)

![](https://githubawesome.com/content/images/2026/02/image-268.png)

**No. 17 — babyshark** [*https://github.com/vignesh07/babyshark*](https://github.com/vignesh07/babyshark?ref=githubawesome.com)

![](https://githubawesome.com/content/images/2026/02/image-269.png)

**No. 18 — Lifo** [*https://github.com/lifo-sh/lifo*](https://github.com/lifo-sh/lifo?ref=githubawesome.com)

![](https://githubawesome.com/content/images/2026/02/image-270.png)

Lifo is a Linux-like operating system that runs in the browser. Instead of emulating Linux, it uses browser APIs as system calls and the browser runtime as the kernel. It includes a bash-like shell with a lexer, parser, and AST interpreter. The shell supports pipes, variable expansion, and job control. It has over 60 commands covering filesystem operations, text processing, networking, and archiving.
