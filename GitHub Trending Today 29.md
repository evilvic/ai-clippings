This is GitHub Trending Today, episode 29. and we have a rapid-fire list of 34 trending open source projects on GitHub.

![](https://www.youtube.com/watch?v=ZWE-3QsH17w)

No. 1 - Coasts

![](https://githubawesome.com/content/images/2026/03/image-473.png)

Giving autonomous agents full host access is a security risk most people accept by default because the alternative has always been complicated. Coasts is localhost service isolation and orchestration for Git worktrees, built by a former YC founder. Agents spin up environments, run servers, and execute code inside containerized hosts without touching the main system.

[https://github.com/coast-guard/coasts](https://github.com/coast-guard/coasts?ref=githubawesome.com)

No. 2 - Agent Flow

![](https://githubawesome.com/content/images/2026/03/image-474.png)

Claude Code running in the terminal gives you output but no visibility into what's actually happening between the lines. Agent Flow connects to your Claude sessions and renders the entire execution as an interactive node graph in real time. Watch agents think, branch, spawn sub-agents, and execute tool calls as they happen, in your browser or VS Code.

[https://github.com/patoles/agent-flow](https://github.com/patoles/agent-flow?ref=githubawesome.com)

No. 3 - Hermes Agent

![](https://githubawesome.com/content/images/2026/03/image-475.png)

Hermes Agent from Nous Research is an autonomous agent built on the Hermes model with a built-in learning loop. It creates skills from experience, persists knowledge into long-term memory, and compounds that learning over time. Host it on a $5 VPS and interact with it directly through Telegram while it works in the background. An agent that actually gets smarter the longer you run it.

[https://github.com/NousResearch/hermes-agent](https://github.com/NousResearch/hermes-agent?ref=githubawesome.com)

No. 4 - Another

![](https://githubawesome.com/content/images/2026/03/image-476.png)

Another mirrors and controls Android devices over USB, built with Tauri, React, and Rust. The MCP server is what makes it interesting for developers: Claude Code or Cursor can connect to it and autonomously click, scroll, type, and control your physical Android device directly from the terminal. AI-driven Android automation without emulators or complex ADB scripting.

[https://github.com/Zfinix/another](https://github.com/Zfinix/another?ref=githubawesome.com)

No. 5 - Heerich

![](https://githubawesome.com/content/images/2026/03/image-477.png)

Heerich renders voxel scenes directly to SVG with zero dependencies. Boolean operations, oblique and perspective cameras, crisp mathematical 3D geometry in the browser without touching WebGL. Inspired by Erwin Heerich's geometric cardboard sculptures, which is exactly the aesthetic the output delivers. For web projects that need clean 3D geometry without the WebGL overhead, this is the lightweight path.

[https://github.com/meodai/heerich](https://github.com/meodai/heerich?ref=githubawesome.com)

No. 6 - There Is No Spoon

![](https://githubawesome.com/content/images/2026/03/image-478.png)

Most ML resources assume you either already know the math or are willing to grind through it from scratch. There Is No Spoon takes a third path: machine learning from first principles, explained through software engineering mental models. Backpropagation, transformers, diffusion models, all demystified using the intuitions you already have as a developer.

[https://github.com/dreddnafious/thereisnospoon](https://github.com/dreddnafious/thereisnospoon?ref=githubawesome.com)

No. 7 - Notchy

![](https://githubawesome.com/content/images/2026/03/image-479.png)

The MacBook notch sits there doing nothing while your AI agents run invisibly in the background. Notchy is an open-source foundation that turns it into a live dashboard. Developers are already building on top of it — Claw'd Notch uses it to drop down real-time status on background AI coding agents across multiple repositories. Dead screen real estate, actually earning its keep.

[https://github.com/adamlyttleapps/notchy](https://github.com/adamlyttleapps/notchy?ref=githubawesome.com)

No. 8 - claude-token-efficient

![](https://githubawesome.com/content/images/2026/03/image-480.png)

Claude-token-efficient is a single CLAUDE.md file you drop into your project root. No code changes, no wrapper, no proxy. It sets strict system-level rules: no redundant context, no speculative features, no long intros, no fluff. Return the simplest working solution and stop. Benchmarks show up to 63% reduction in output verbosity. The file adds a small input token cost per prompt, but for high-volume agent loops the output savings cover it immediately.

[https://github.com/drona23/claude-token-efficient](https://github.com/drona23/claude-token-efficient?ref=githubawesome.com)

No. 9 - Pretext

![](https://githubawesome.com/content/images/2026/03/image-481.png)

For twenty years, we've let CSS handle text layout. But complex animations cause massive performance lag. Meet Pretext! It throws CSS completely out the window. It uses pure JavaScript and arithmetic to measure text, completely bypassing the DOM. The result? Insane sixty-frames-per-second fluid text animations with zero layout lag!

[https://github.com/chenglou/pretext](https://github.com/chenglou/pretext?ref=githubawesome.com)

No. 10 - Hucre

![](https://githubawesome.com/content/images/2026/03/image-482.png)

Generating spreadsheets in a web app usually means a backend service or a heavy library with a long dependency chain. Hucre is a zero-dependency spreadsheet engine in pure TypeScript that runs everywhere. Reads and writes XLSX, CSV, and ODS, with full support for cell styles, tables, conditional formatting, and rich text. Client-side spreadsheet generation with no backend required.

[https://github.com/productdevbook/hucre](https://github.com/productdevbook/hucre?ref=githubawesome.com)

No. 11 - MolmoWeb

![](https://githubawesome.com/content/images/2026/03/image-483.png)

Most web scraping agents are one HTML restructure away from breaking completely. MolmoWeb ignores the HTML entirely. It navigates the web purely from screenshots, understanding the UI visually the way a human does, then clicks, scrolls, and types based on what it sees. No DOM dependency, no fragile selectors. When the HTML changes, it doesn't care.

[https://github.com/allenai/molmoweb](https://github.com/allenai/molmoweb?ref=githubawesome.com)

No. 12 - Tome

![](https://githubawesome.com/content/images/2026/03/image-484.png)

Corporate meeting bots recording your conversations and storing them on someone else's servers is a legitimate concern that most people just accept. Tome captures both system and microphone audio locally, transcribes the entire meeting offline, and saves the formatted output directly into your Obsidian vault. No cloud, no API keys, nothing leaving your machine.

[https://github.com/Gremble-io/Tome](https://github.com/Gremble-io/Tome?ref=githubawesome.com)

No. 13 - Codex Plugin CC

![](https://githubawesome.com/content/images/2026/03/image-485.png)

OpenAI just dropped an official plugin for Claude Code. Codex-plugin-cc lets you invoke OpenAI's Codex model directly inside your Claude Code terminal. Just had Claude write a massive feature? Run slash codex review to have Codex double-check the work. Want to go harder? Use adversarial review to make Codex act like a ruthless senior engineer and aggressively pressure-test your code before merging! Two rival AIs, one terminal.

[https://github.com/openai/codex-plugin-cc](https://github.com/openai/codex-plugin-cc?ref=githubawesome.com)

No. 14 - Phantom

![](https://githubawesome.com/content/images/2026/03/image-486.png)

Phantom runs continuously on its own virtual machine as a dedicated AI co-worker built on the Claude Agent SDK. Persistent vector memory, Slack integration, and a self-rewriting configuration that improves after every session. It doesn't stop when you close the terminal because it was never running in your terminal to begin with. A co-worker that's always on, always learning, running on its own machine.

[https://github.com/ghostwright/phantom](https://github.com/ghostwright/phantom?ref=githubawesome.com)

No. 15 - Spool

![](https://githubawesome.com/content/images/2026/03/image-487.png)

Every developer has a graveyard of useful code snippets, saved links, and agent outputs they can never find when they actually need them. Spool indexes your Claude Code logs, Codex history, GitHub stars, and Twitter bookmarks into a single local search bar on Mac. Everything you've generated or saved, instantly searchable from one place. The search engine for your own digital output.

[https://github.com/spool-lab/spool](https://github.com/spool-lab/spool?ref=githubawesome.com)

No. 16 - Learn Docker and K8s

![](https://githubawesome.com/content/images/2026/03/image-488.png)

Docker and Kubernetes documentation assumes you already know what you're doing, which makes starting from scratch genuinely difficult. Learn Docker and K8s strips the enterprise jargon and walks through containerization and orchestration as a clear step-by-step learning path. Package, deploy, and scale applications using modern DevOps practices, explained for people who are actually learning it for the first time.

[https://github.com/ericboy0224/learn-docker-and-k8s](https://github.com/ericboy0224/learn-docker-and-k8s?ref=githubawesome.com)

No. 17 - AI Website Cloner Template

![](https://githubawesome.com/content/images/2026/03/image-489.png)

If you've ever seen a beautifully designed website and wanted to build exactly that, this Claude Code skill is magic. You point it at a URL, and it dispatches parallel AI builder agents into separate Git worktrees to reconstruct every single section of the site simultaneously! It automatically downloads assets and rebuilds the site into a pixel-perfect Next.js app in minutes!

[https://github.com/JCodesMore/ai-website-cloner-template](https://github.com/JCodesMore/ai-website-cloner-template?ref=githubawesome.com)

No. 18 - ColaMD

![](https://githubawesome.com/content/images/2026/03/image-490.png)

Typora and most Markdown editors treat the file as yours alone — close it, reopen it to see external changes. ColaMD watches the filesystem and live-refreshes instantly as background AI agents write to the file. No manual reloading, no stale views while an agent is mid-edit. A Markdown editor built for the reality of having an AI writing to your files in the background.

[https://github.com/marswaveai/ColaMD](https://github.com/marswaveai/ColaMD?ref=githubawesome.com)

No. 19 - OfficeCLI

![](https://githubawesome.com/content/images/2026/03/image-491.png)

AI agents that need to work in real business environments eventually hit the wall of proprietary Office formats. OfficeCLI is a single binary CLI written in C# that lets agents read, edit, and automate Word, Excel, and PowerPoint files natively from the terminal. Generate spreadsheets, rewrite contracts, scrape data from pitch decks, all without Microsoft Office installed on the host machine.

[https://github.com/iOfficeAI/OfficeCLI](https://github.com/iOfficeAI/OfficeCLI?ref=githubawesome.com)

No. 20 - Sentrux

![](https://githubawesome.com/content/images/2026/03/image-492.png)

Sentrux is a real-time architectural sensor written in Rust that visualizes your codebase as a live interactive treemap, catching structural degradation as it happens. The MCP server integration is the key detail: your agent can query Sentrux directly, see how messy its own code is getting, and course-correct before the architecture becomes a problem to untangle.

[https://github.com/sentrux/sentrux](https://github.com/sentrux/sentrux?ref=githubawesome.com)

No. 21 - Harness

![](https://githubawesome.com/content/images/2026/03/image-493.png)

Harness is a meta-skill for Claude Code that builds the agent team for you. Tell it to build a harness for your project, and it autonomously designs the multi-agent architecture, generates specialized worker agents, defines the pipeline structure — fan-out, supervisor, or whatever fits the problem — and equips each agent with the skills it needs. You describe the problem. It designs the team that solves it.

[https://github.com/revfactory/harness](https://github.com/revfactory/harness?ref=githubawesome.com)

No. 22 - LeWorldModel

![](https://githubawesome.com/content/images/2026/03/image-494.png)

LeWorldModel is the official codebase for a Stable End-to-End Joint-Embedding Predictive Architecture from Pixels. Instead of predicting raw pixel outputs, it predicts abstract representations of future states, pushing toward AI systems that build genuine internal world models. The distinction matters for both computer vision and reinforcement learning research.

[https://github.com/lucas-maes/le-wm](https://github.com/lucas-maes/le-wm?ref=githubawesome.com)

No. 23 - PrismerCloud

![](https://githubawesome.com/content/images/2026/03/image-495.png)

AI agents that can't remember previous sessions and can't communicate with each other are fundamentally limited in what they can accomplish together. PrismerCloud is a self-hosted server that gives agents persistent memory, knowledge that evolves over time, and a real-time WebSocket messaging system. Independent agents discover each other and coordinate directly to solve problems collaboratively.

[https://github.com/Prismer-AI/PrismerCloud](https://github.com/Prismer-AI/PrismerCloud?ref=githubawesome.com)

No. 24 - Atomic Chat

![](https://githubawesome.com/content/images/2026/03/image-496.png)

Atomic Chat is a native desktop AI assistant built on the OpenClaw framework with no Docker configuration required. Plug in your API keys or connect local Hugging Face models, and you get a private ChatGPT-style interface that can also run terminal skills. Clean install, no cloud dependency, your conversations stay on your machine. The ChatGPT experience without the ChatGPT data policies.

[https://github.com/AtomicBot-ai/Atomic-Chat](https://github.com/AtomicBot-ai/Atomic-Chat?ref=githubawesome.com)

No. 25 - Dither Playground

![](https://githubawesome.com/content/images/2026/03/image-497.png)

Dither Playground converts any image into thousands of interactive dithered particles that respond to your cursor in real time. Hover to push them around, click to scatter the whole thing into a physics-driven swarm. Runs entirely in the browser, and the effect is the kind of thing you immediately want to steal for a landing page hero section.

[https://github.com/lenxism/dither](https://github.com/lenxism/dither?ref=githubawesome.com)

No. 26 - figma-mcp-go

![](https://githubawesome.com/content/images/2026/03/image-498.png)

Figma's REST API rate limits hit hard when an AI agent is making thousands of calls during active UI development. Figma-mcp-go bypasses the API entirely by connecting through a local plugin bridge instead of API keys. Zero rate limits, real-time read and write access, and your agent can extract JSON from selected nodes, clone components, and iterate directly on the canvas. Runs via NPX with no configuration or local Go installation required.

[https://github.com/vkhanhqui/figma-mcp-go](https://github.com/vkhanhqui/figma-mcp-go?ref=githubawesome.com)

No. 27 - HolyClaude

![](https://githubawesome.com/content/images/2026/03/image-499.png)

Setting up Claude Code on a VPS usually means wrestling with dependencies, display issues, and browser sandbox crashes. HolyClaude is a single Docker Compose file that eliminates all of that. Claude Code CLI, a browser-accessible terminal UI, five AI CLIs, and 50+ development tools pre-installed. Your agent can spin up a web app, navigate to it, take screenshots, and run visual tests without the container falling over.

[https://github.com/coderluii/holyclaude](https://github.com/coderluii/holyclaude?ref=githubawesome.com)

No. 28 - Lark CLI

![](https://githubawesome.com/content/images/2026/03/image-500.png)

The official Lark CLI covers Messenger, Docs, Sheets, and Calendar with over 200 commands from the terminal. The 19 built-in AI agent skills are what make it interesting: your local AI can read corporate emails, schedule meetings, and update spreadsheets autonomously without you touching the Lark interface. Full Lark ecosystem automation, driven directly from your terminal.

No. 29 - Usecomputer

![](https://githubawesome.com/content/images/2026/03/image-501.png)

Python wrappers for desktop automation add latency that compounds fast when an agent is making dozens of sequential actions. Usecomputer is a native Zig CLI that gives AI agents low-latency control over macOS and Linux desktops. Accessibility snapshots, mouse control, typing, scrolling, all available programmatically with minimal overhead.

[https://github.com/remorses/usecomputer](https://github.com/remorses/usecomputer?ref=githubawesome.com)

No. 30 - Create Context Graph

Create Context Graph does for graph-powered AI apps what create-next-app does for React projects. Run the CLI, and in under five minutes you have a complete FastAPI backend with an AI agent that uses a Neo4j knowledge graph for memory. Complex entity relationships, persistent across sessions, without building the infrastructure from scratch.

[https://github.com/neo4j-labs/create-context-graph](https://github.com/neo4j-labs/create-context-graph?ref=githubawesome.com)

No. 31 - Claude Watch

![](https://githubawesome.com/content/images/2026/03/image-503.png)

Kicking off a long Claude Code session and walking away means being tethered to your laptop for approval prompts. Claude Watch pushes those prompts directly to your Apple Watch. Agent needs permission to run a terminal command, your wrist gets the notification, you approve it on the spot. AI agent oversight from anywhere in the house.

[https://github.com/shobhit99/claude-watch](https://github.com/shobhit99/claude-watch?ref=githubawesome.com)

No. 32 - vLLM TurboQuant

![](https://githubawesome.com/content/images/2026/03/image-504.png)

The TurboQuant algorithm is now integrated directly into vLLM. That means the 3-bit/2-bit KV cache compression is available in the industry-standard inference engine without any custom setup. Deploy massive MoE models like Qwen 3.5 on existing GPU hardware with double the context capacity and no changes to your serving infrastructure.

[https://github.com/mitkox/vllm-turboquant](https://github.com/mitkox/vllm-turboquant?ref=githubawesome.com)

No. 33 - OpenClaw Auto Dream

![](https://githubawesome.com/content/images/2026/03/image-505.png)

OpenClaw Auto Dream adds an automated task runner and CI layer to the OpenClaw ecosystem. Agents pick up complex tasks, work through the architectural solutions autonomously, and execute reliably in the background. For anyone running a serious OpenClaw workflow who needs their agents operating continuously without manual hand-holding between tasks, this is the piece that closes the loop.

[https://github.com/LeoYeAI/openclaw-auto-dream](https://github.com/LeoYeAI/openclaw-auto-dream?ref=githubawesome.com)

No. 34 - Zinc

![](https://githubawesome.com/content/images/2026/03/image-506.png)

Zinc is an LLM inference engine written in Zig, built specifically for RDNA3 and RDNA4 via Vulkan. Hand-tuned for AMD's memory hierarchy with wave64 dispatch and architecture-aware tiling, it hits over 90% of theoretical memory bandwidth on the matrix multiplications that dominate LLM decoding. 35-billion parameter models running comfortably on $550 consumer AMD hardware.

[https://github.com/zolotukhin/zinc](https://github.com/zolotukhin/zinc?ref=githubawesome.com)
